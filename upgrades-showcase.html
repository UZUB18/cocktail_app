From Speakeasy to Supernova

01  The Starting Point

The app today is a premium local-first cocktail companion — React 19 + TypeScript + Vite, localStorage persistence, 50+ catalog specs, and a warm amber ✦ speakeasy theme. Beautiful in its category but running a single 4,700-line CSS file, hash-routing via window.location.hash, and all views in one App.tsx.

Current Architecture at a Glance

  • Routing — Manual hash-based (window.location.hash). No React Router. Views toggled with activeView state + conditional rendering.
  • State — Single useState for StoredState. Every view gets props drilled through the tree. No context, no store.
  • Styling — One giant App.css (4,744 lines). No CSS modules, no CSS-in-JS, no Tailwind.
  • Images — Dynamic SVG fallbacks for cocktails without photos. Generated per-cocktail via hashseed.
  • Code splitting — None. All views load in one bundle despite being toggled.
The app's bones are excellent — local-first, private, zero-server, deep cocktail knowledge. The upgrades aren't about fixing broken things. They're about unlocking what's already there with better architecture, richer interactions, and design that leaves an impression.

05  Frontend Architecture

These aren't user-facing, but they enable everything else. The current monolith works for a single-user local app, but it's at the limit of maintainability.

ARCH

React Router v7 + Code Splitting

Replace window.location.hash with React Router. Each view becomes a lazy-loaded route via React.lazy() + Suspense. The initial bundle drops from ~160KB to ~40KB. Deep-linking works natively. Back/forward navigation without hacks.

// Before
const activeView = useState<View>(initialView)
{activeView === 'Recipes' && <RecipeDetail />}

// After
const RecipeDetail = React.lazy(() => import('./views/RecipeDetail'))
<Route path="/recipe/:id" element={<RecipeDetail />}/>
ARCH

State Management: Zustand

Extract StoredState into a Zustand store. Eliminates prop drilling through 10+ view components. Built-in persistence middleware replaces manual localStorage calls. Subscribe to slices instead of re-rendering the entire tree on every change.

ARCH

CSS Architecture: Modules or Tailwind

4,744 lines in one file is the limit. Split into per-component CSS modules, or adopt Tailwind v4 for utility-first styling. Either approach eliminates selector collisions, makes dead code detectable, and allows themed component libraries.

ARCH

Route-Based View Splitting

Each of the 10 views (HomeView, SearchView, RecipeDetail, etc.) currently lives in App.tsx. Extract into separate files under src/views/. Shared components (sidebar, topbar, ingredient row, flavor meters) into src/components/.

ARCH

PWA — Installable & Offline

Add a service worker + manifest. The app is already 100% offline-capable (localStorage + no server calls). Making it a PWA means users install it on their phone home screen, get a splash screen, and never see a browser chrome. This is a 20-minute change for a massive UX win.

⚡ Effort: Low🎯 Payoff: Very High

06  New Features & Capabilities

FEATURE

Cocktail Randomizer

"Surprise Me" button that picks a random cocktail from what you can make with current inventory. Adjustable constraints: "something with rum" or "under 4 ingredients". Perfect for decision paralysis nights.

FEATURE

Voice-Through Method Mode

A "voice mode" in Practice Lab that reads the method steps aloud (Web Speech API) with touch-forward progression. Hands-free cocktail making. Your phone becomes a bar assistant that doesn't need you to look at it.

FEATURE

Recipe Sharing via Encoded URL

Generate a shareable URL that encodes the recipe spec (compressed, base64). Someone opens it, the app detects the payload and imports it. No server, no backend — just URLs as data carriers.

FEATURE

Smart Substitution Engine

Currently substitutions are static text. Upgrade to a rule-based engine: "No Campari? Try Aperol (lighter) or Gran Classio (rounder)." Show ABV impact, sweetness delta, and color change preview.

FEATURE

Barcode Scanner for Inventory

Scan a bottle's barcode via device camera (using navigator.mediaDevices) to auto-add it to inventory. Leverage open product databases. Transforms inventory from a typing chore into a 2-second action.

FEATURE

Nutrition & ABV Calculator

For each cocktail, show estimated ABV, calories, sugar content based on ingredient specs. Batch planner already scales ingredients — plugging ABV math on top is trivial.

FEATURE

Party Host View

Full-screen mode showing the party menu with large text, countdown timers between rounds, "next pour" suggestions. Optimized for a bartender who doesn't want to squint at their phone while holding a shaker.

FEATURE

Technique Reference Library

Embedded mini-guides for techniques: proper shaking mechanics, stirring technique, egg white dry shake, swizzling, muddling. Expandable accordion in Practice Lab. Video/animated diagram support.

FEATURE

Cocktail History Calendar

Track what you've made and when. Calendar heatmap showing your cocktail activity. "You made 12 Negronis this month" — a mix of utility and gentle self-awareness.

FEATURE

Ingredient Expiry Tracking

For perishables (citrus, syrups, vermouth, cream), set an expiry date on addition. The app warns when something is past its prime. Critical for anyone who's ever tasted week-old lime juice.

The best upgrade isn't a feature. It's a feeling. A cocktail app should make you want to make drinks — not manage data. Every interaction should pull you toward the shaker.

07  UX Polish — The Small Things

These aren't features you market. They're the difference between an app you use and an app you love. Each one is low-to-medium effort, and together they compound into a fundamentally better experience.

UX

Drag-and-Drop Ingredient Reordering

In the Recipe Creator, let users drag ingredients to reorder them. Currently it's delete-and-retype. Native HTML5 drag + drop, ~50 lines of code.

UX

Global Keyboard Shortcuts

⌘K is there. Add: ⌘N new recipe, ⌘S save, ⌘E export, ⌘I inventory, ⌘R randomizer.

UX

Undo/Redo for Recipe Editing

History stack for custom recipe creation. Accidentally removed an ingredient? ⌘Z brings it back. Reduces the fear of experimentation.

UX

Rich Command Palette

The search bar already has ⌘K. Upgrade it to a full command palette: open any view, search cocktails, toggle settings, run actions. Inspired by Linear/VS Code. Power users never touch the sidebar.

UX

First-Run Onboarding Flow

When localStorage is empty (first visit), show a 3-step intro: add your first 3 bottles, browse the catalog, save your first spec. Reduces the empty-state drop-off. The app currently drops users straight into demo data, which works but misses the "aha" moment.

UX

Empty States with Purpose

Every empty list (no search results, no inventory, no practice logs) shows a contextual illustration + CTA — not just "nothing here." Make the user feel guided, not abandoned.

UX

Print-Ready Recipe Cards

CSS @media print styles that render a recipe as a clean card: ingredients on the left, method on the right, QR-code link back to the app. Users who print specs for their bar book will thank you.

UX

Touch-Optimized Mobile Navigation

Bottom tab bar on mobile replaces the sidebar slide-out. Swipe gestures to go back, long-press ingredients for context menu. Make the phone experience feel native.

08  Stretch Goals — The Long Game

VISION

AI-Assisted Spec Balancing

Describe what you want: "Something tart with mezcal and a hint of smoke, 20% ABV." The app suggests a balanced spec with calculated dilution. Leverages the existing cocktail data as training.

VISION

Guest Profiles & Preferences

Save guest preferences (likes: rum, dislikes: Campari, allergic: eggs). When a guest is selected, the app highlights cocktails they'd enjoy and warns about ingredients to avoid. For hosts.

VISION

Multi-Device Sync via WebRTC

Peer-to-peer sync between devices without a server. Open the app on your phone, scan a QR code from your laptop, your inventory syncs. WebRTC data channels + local encryption. No backend needed.

VISION

Ingredient Marketplace / Curated Shopping

For complex recipes, show a "build this spec" shopping list with links to recommended bottles, syrups, and bar tools. Could be affiliate-linked or just informational.

🍸 The Verdict

Cocktail Colleague has excellent foundations — a rich catalog, solid local-first architecture, and a premium aesthetic. The upgrades aren't about fixing what's broken. They're about unlocking the sensory, architectural, and experiential potential of an app that already does the hard part right.

Start with the Quick Wins (PWA, keyboard shortcuts, spring buttons) — they're trivial to implement and immediately visible. Then tackle the Foundation Builds (Router, store, file splitting) — they make everything else faster to build. Then pick one Differentiator that matches your vision.

The gap between "good cocktail app" and "cocktail app you show people" is smaller than you think.
